## The string-mode pattern (used in Tasks 4 and 5)

Each `output=` op currently takes `std::optional<int> output` and validates a range. Convert to a string, parsed in the constructor with a static helper (the same shape as `parse_fill` in the backtest engines), keeping the internal integer logic unchanged. Example skeleton:

```cpp
static int parse_output(const std::string& s) {
    if (s == "cleaned") return 0;
    if (s == "flag")    return 1;
    if (s == "nan")     return 2;
    throw std::invalid_argument("output must be \"cleaned\", \"flag\", or \"nan\".");
}
```
The constructor takes `const std::string& output = "<default>"`, stores `output_ = parse_output(output)`, and the rest of the op is untouched. The binding's `py::arg("output")` default becomes the string.
